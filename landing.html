What it is

Claude & friends generate great HTML. hart gives it a home — a real, shareable, sandboxed URL — for any terminal agent, not just one vendor's.

publish

HTML or JSX → a URL

Pipe a self-contained file; get /a/owner/name back. JSX is transpiled in-browser with a same-origin React runtime — no build step, no CDN.

versioned

v1 · v2 · latest

Re-publish to append a version; latest always tracks newest, old versions stay immutably pinned, and rollback reverts instantly.

sandboxed

Hostile pages stay inert

Every artifact is served under a strict CSP — no external scripts, no network. A publish-time linter blocks anything that reaches off-page.

agent-first

A CLI, not a dashboard

JSON in, JSON out, semantic exit codes, no UI tax. The interface is the CLI — an agent drives it blind.

The template + data model

Publish a template once. Then push just the data — the artifact re-renders. An agent keeps a page live as its source data changes.

01 — template (once)

Push the shape

Use {{placeholders}} or read window.HART_DATA from your script.

# a greeting template
echo '<h1>Hello, {{name}}!</h1>' > g.html
hart publish g.html --owner acme --artifact hello
02 — data (as it changes)

Push the values

Same URL, new content. No re-uploading the page.

hart data acme/hello '{"name":"Ada"}'
# → /a/acme/hello now says "Hello, Ada!"
hart data acme/hello '{"name":"Bob"}'
# → "Hello, Bob!" — one call

Run your own

One binary is the CLI and the host. Free & open by default; set a token to gate it. Rate-limited so you can leave it open.

# build & run the host (SQLite store, your box)
./build.sh && ./hart serve 8799

# publish from anywhere
export HART_URL=https://hart.yourdomain
hart publish report.html --owner you --artifact q3

# knobs: HART_TOKEN (gate publishing) ·
# HART_MAX_SUBMITS_PER_MIN (10) · HART_MAX_OWNER_MB (30)

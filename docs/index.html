What it is

The agent-first artifact host, at a glance.

A home for agent HTML

Claude and friends generate great HTML. hart gives it a real, shareable, sandboxed URL — for any terminal agent, not one vendor's.

Pipe a file, get a URL

Pipe a self-contained file; get /a/owner/name back. JSX is transpiled in-browser with a same-origin React runtime — no build step, no CDN.

Immutable versions

Re-publish to append a version. Latest always tracks newest; old versions stay immutably pinned; rollback reverts instantly.

Sandboxed by default

Every artifact is served under a strict CSP — no external scripts, no network. A publish-time linter blocks anything that reaches off-page.

Agent-first I/O

JSON in, JSON out, semantic exit codes, no UI tax. The interface is the CLI — an agent drives it blind.

Content-addressed

Publishes are hash-addressed and deduplicated. The store is just the filesystem — no index, schedule with plain cron.

Operator visibility

Run one instance for all your agents. An admin token unlocks cross-owner discovery — list every owner and artifact, and move/rename them in place (history + read-key preserved).

Living deliverables

A page can stay alive: push new data to the same URL, or let hart self-refresh it on a schedule from a URL or command. Opt a page into live mode and it repaints in the browser without a reload.

hart Pro (open-core)

The CLI is fully free; a license key unlocks higher storage limits, an audit log (who changed what), and teams. Honest, offline-verified keys — buy agent-first with hart upgrade.

Teams & SSO

Share a namespace with per-member keys, each with its own identity in the audit log. Members self-onboard with hart join via any OpenID Connect provider you point it at — you bring the IdP.

The template + data model

Separate the page from its data; update either independently.

Publish a template once, then push just the data — the artifact re-renders. An agent keeps a page live as its source data changes. Use double-brace placeholders, or read window.HART_DATA from your script.

1 - publish a template
echo '<h1>Hello, {{name}}</h1>' > g.html
hart publish g.html --owner acme --artifact hello
# -> /a/acme/hello
2 - push data (one call, re-renders live)
hart data acme/hello data.json
# data.json holds the values; /a/acme/hello now greets by name
3 - or let hart keep it fresh itself
hart refresh acme/hello --url https://api/metrics --every 15m
hart live acme/hello on   # repaint in the browser, no reload

FAQ

How is this different from Netlify or Pastebin?+

Netlify is human-first (a dashboard, a build). Pastebin does not render or sandbox. hart is agent-first (JSON/CLI, no UI) and safely hosts UNTRUSTED agent HTML under a strict CSP with content-addressed versioning.

Do I need Node or a build step?+

No. hart is one machin (MFL) static binary. JSX is transpiled in the browser with a same-origin React runtime — no bundler, no CDN.

Can I roll back a bad publish?+

Yes. Every publish is an immutable version; latest tracks the newest, old versions stay pinned at /a/owner/name/vN, and rollback is instant.

Who is it for?+

Any terminal agent — Cursor, aider, Codex, Gemini CLI, or your own — that generates HTML and needs a live, shareable, safe URL back.
